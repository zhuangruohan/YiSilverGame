## 1. 总体推进原则

### 1.1 现在不要继续做的事

当前不要继续扩大工作面：

- 不要继续精修 7 张地图的所有细节。
- 不要一次性接完整任务系统。
- 不要一次性做背包、图鉴、结局、JSON、完整剧情树。
- 不要再让 Codex 同时修改地图、NPC、跳转、小游戏、任务系统。
- 不要直接 `git add .`。
- 不要移动 `assets/maps/`、`assets/images/`、`assets/tilesets/` 下的资源。

### 1.2 之后每个 Codex 任务的固定节奏

```text
1. 只给一个小任务。
2. 明确允许修改哪些文件。
3. 明确禁止做哪些功能。
4. 要求 Codex 输出修改文件、自测步骤、是否影响旧功能。
5. 本地真实窗口测试通过后再 commit。
6. 一个任务一个 commit。
```

### 1.3 当前主线闭环目标

最终 Demo 最少要形成这条链：

```text
进入游戏
→ 在 village_hub 探索
→ 与 NPC 对话
→ 进入其他场景或核心玩法入口
→ 完成拓印
→ 完成敲银修复
→ 回到 NPC / 主场景显示任务完成
```

这条闭环比“7 张地图都能跑”更重要。

### Task 1.1：修复真实窗口键盘输入与场景跳转

#### 当前现象

- 画面左上角显示 `Nearby Exit: exit_to_xxx`。
- 画面显示“按 E 进入”。
- 说明出口检测成功。
- 但按 E 不跳转。
- 按数字键 1/2/3/4 也不强制切图。

#### 判断

这不是地图问题，而是代码输入事件处理问题：

```text
Game.handle_events() / Scene.handle_event() / pygame.key.get_pressed() 兜底输入
这条链路没有稳定触发。
```

#### 允许修改

- `src/game.py`
- `src/scene.py`

#### 禁止修改

- 不要修改 Tiled 地图对象。
- 不要新增出口。
- 不要移动 TMX / TSX / PNG。
- 不要实现小游戏、任务、背包、JSON、图鉴、结局。

#### Codex 指令摘要

```text
请只修复真实窗口中按 E、按数字键 1/2/3/4 都没有触发场景切换的问题。

当前画面已经能显示 Nearby Exit 和“按 E 进入”，说明地图出口读取成功，不要改地图。

重点修改：
1. src/game.py 中打印所有 KEYDOWN：
   [Game] KEYDOWN: key_name
2. PLAYING 状态下把所有事件传给 self.scene.handle_event(event)。
3. src/scene.py 中一进入 handle_event 就打印：
   [Scene] KEYDOWN received: key_name
4. 数字键 1/2/3/4 强制 load_scene：
   1 -> village_hub
   2 -> village_market
   3 -> workshop_exterior
   4 -> river_valley
5. E 键非对话状态下优先处理 nearby_exit，再处理 NPC。
6. 增加 pygame.key.get_pressed() 兜底输入，做边缘触发：
   [FallbackInput] E pressed
   [FallbackInput] force load scene: xxx
7. 保留 transition_cooldown，避免刚跳过去又触发返回。
8. debug overlay 缩小，并支持 F3 显示 / 隐藏。

真实窗口自测：
- 按 2 能切到 village_market。
- 按 1 能回 village_hub。
- 靠近出口按 E 能切换。
- 终端必须出现 Game / Scene / FallbackInput 的调试信息。
```

#### 验收标准

- 按 2 能切到 `village_market`。
- 按 1 能回 `village_hub`。
- 靠近出口按 E 能跳转。
- 终端出现 `[Game] KEYDOWN` 或 `[FallbackInput]`。
- NPC 对话不被破坏。
- Esc 仍能退出。

#### 提交建议

```bash
git add src/game.py src/scene.py
git commit -m "fix(input): handle scene transition hotkeys"
git push gitee main
git push github main
```

### Task 1.3：修复玩家左右朝向

#### 当前问题

玩家图片默认朝左，往右走时仍朝左。

#### 允许修改

- `src/player.py`

#### Codex 指令摘要

```text
请只修复 Player 角色方向。
1. 加载原图为 self.image_left。
2. 使用 pygame.transform.flip(self.image_left, True, False) 生成 self.image_right。
3. 增加 facing 属性。
4. 向右移动 facing = "right"。
5. 向左移动 facing = "left"。
6. 上下移动保持上一次左右方向。
7. draw() 根据 facing 选择图片。
8. 不改变碰撞 rect，不做完整动画。
```

#### 验收标准

- 向左走角色朝左。
- 向右走角色朝右。
- 上下走保持上一方向。
- 碰撞、NPC、跳转不受影响。

#### 提交建议

```bash
git add src/player.py
git commit -m "fix(player): update sprite facing direction"
```
