版本：纹样寻源优化版  
适用项目：Python + Pygame 2D 文化修复探索游戏  
适用对象：项目开发者、Codex / ChatGPT 协作开发工具、测试人员、后续维护人员

## 当前实现同步说明（2026-07-01）

本文档是设计规范底稿，包含完整版本规划。当前课程演示版本的真实主线为：

```text
村寨长者对话 -> 集市寻源 -> 河谷影纹净化 -> 节庆广场守护 -> Ending
```

当前真实代码结构以 `src/core`、`src/scenes`、`src/maps`、`src/entities`、`src/systems`、`src/ui`、`src/resources`、`src/minigames` 为准。文中出现的 `src/game.py`、`src/scene.py`、`src/tilemap.py` 等旧路径仅作为历史设计阶段参考。

完整工坊敲银修复、完整图鉴、更多语音、存档、多结局和更复杂纹样玩法均为预留 / 后续扩展，不应误写成当前完整实现。

## 一、文档目的

本文档用于规范《银纹秘境：彝饰守护者》的代码开发、资源管理、地图制作、数据配置、AI 协作、测试验收和提交记录方式。

本项目采用 Python + Pygame + Tiled + PyTMX + JSON 的技术路线，核心玩法围绕“纹样寻源、纹样空间小关卡、拓片获得、敲银修复、银饰套装进度、银光值评价和节庆展示”展开。

开发规范的目标不是限制实现，而是保证：

1. 代码结构清晰；
2. 功能模块可拆分；
3. 地图和数据可配置；
4. Codex 生成代码时不随意扩展；
5. 每个阶段都能运行、能测试、能回滚；
6. 项目最终能形成完整可展示 Demo。

## 二、项目开发总原则

### 2.1 优先级原则

开发顺序必须遵循：

```text
先跑通主流程
再补充特色机制
最后优化美术音效
```

不得在基础流程未完成前大量加入可选扩展。

### 2.2 主流程闭环优先

项目必须优先保证以下流程完整：

```text
启动游戏
→ 主菜单
→ 村寨入口
→ NPC 对话
→ 拾取银饰碎片
→ 前往工坊
→ 获得修复锤
→ 前往山地或集市
→ 完成纹样寻源
→ 获得纹样拓片
→ 收集修复材料
→ 返回工坊
→ 敲银修复
→ 点亮银饰部件
→ 解锁图鉴
→ 前往节庆广场
→ 结局评价
```

### 2.3 功能边界原则

本项目明确不开发以下内容：

1. 复杂战斗系统；
2. 武器和装备升级系统；
3. 商店交易系统；
4. 联网、登录、排行榜；
5. 大型开放世界；
6. 商业级动画系统；
7. 真实图像识别式拓印；
8. 复杂视频剧情系统。

如 AI/Codex 生成了上述内容，应删除或要求重写。

### 2.4 文化表达原则

文化内容应遵守：

1. 不编造复杂民俗细节；
2. 不将民族文化猎奇化；
3. 不写未经证实的绝对寓意；
4. 对纹样来源使用“游戏化转译”表述；
5. 文化说明尽量绑定游戏行为；
6. 图鉴文案短句优先，不写大段论文。

推荐写法：

```text
本游戏将山地轮廓抽象为山纹路径，玩家沿着山脊形路线走完后获得山纹拓片。
```

不推荐写法：

```text
山纹在彝族文化中必然代表某某神话故事。
```

### 4.2 main.py 规范

`main.py` 只负责启动游戏，不应堆功能逻辑。

推荐写法：

```python

def main() -> None:
    game = Game()
    game.run()

    main()
```

不推荐：

```python
# 不要把地图、玩家、任务、背包、图鉴、修复小游戏全部写在 main.py 里
```

## 五、Python 代码规范

### 5.1 命名规范

| 类型 | 规范 | 示例 |
|---|---|---|
| 文件名 | 小写 + 下划线 | `task_manager.py` |
| 类名 | 大驼峰 | `TaskManager` |
| 函数名 | 小写 + 下划线 | `load_scene()` |
| 变量名 | 小写 + 下划线 | `current_scene` |
| 常量名 | 全大写 + 下划线 | `SCREEN_WIDTH` |
| JSON id | 小写 + 下划线 | `mountain_pattern_rubbing` |

### 5.2 文件职责规范

每个文件只负责一类功能。

| 文件 | 职责 |
|---|---|
| `game.py` | 主循环、状态切换、全局模块管理 |
| `scene.py` | 场景加载、场景对象管理 |
| `tilemap.py` | TMX 地图读取、图层绘制、对象层读取 |
| `camera.py` | 摄像机跟随、边界限制、坐标转换 |
| `player.py` | 玩家移动、碰撞、交互 |
| `npc.py` | NPC 显示、对话触发 |
| `item.py` | 可拾取物品 |
| `inventory.py` | 背包数据和背包界面 |
| `task_manager.py` | 任务状态和任务推进 |
| `codex.py` | 银饰图鉴和纹样图鉴 |
| `pattern_source.py` | 纹样寻源入口 |
| `pattern_realm.py` | 走纹样迷宫和检查点 |
| `repair_minigame.py` | 材料检查、敲银判定、修复结果 |
| `score_manager.py` | 银光值、套装进度、结局评价 |

### 5.3 函数规范

函数应尽量短小，每个函数只做一件事。

推荐：

```python
def add_item(self, item_id: str) -> None:
    if item_id not in self.items:
        self.items.append(item_id)
```

不推荐：

```python
# 一个函数里同时处理移动、碰撞、对话、任务、背包、绘制
```

### 5.4 注释规范

必须注释：

1. 坐标转换逻辑；
2. 任务状态推进逻辑；
3. 对象层读取逻辑；
4. 纹样检查点判定逻辑；
5. 敲银判定逻辑；
6. 银光值计算逻辑。

不需要注释过于简单的语句。

## 六、Pygame 主循环规范

### 6.1 主循环结构

推荐主循环拆成三段：

```python
while self.running:
    self.handle_events()
    self.update(dt)
    self.draw()
```

禁止在 `draw()` 中修改任务、背包、图鉴等游戏状态。

### 6.2 事件处理规范

`handle_events()` 负责：

1. 退出事件；
2. 键盘按下事件；
3. 鼠标点击事件；
4. 状态切换请求。

### 6.3 更新逻辑规范

`update(dt)` 负责：

1. 玩家移动；
2. 碰撞检测；
3. 摄像机更新；
4. 当前场景对象更新；
5. 小游戏状态更新。

### 6.4 绘制逻辑规范

`draw()` 负责：

1. 清屏；
2. 绘制地图；
3. 绘制对象；
4. 绘制玩家；
5. 绘制 UI；
6. 刷新屏幕。

## 七、游戏状态规范

### 7.1 状态枚举

建议使用统一状态管理。

```python
from enum import Enum


class GameState(Enum):
    MENU = "menu"
    INTRO = "intro"
    PLAYING = "playing"
    DIALOGUE = "dialogue"
    INVENTORY = "inventory"
    CODEX = "codex"
    PATTERN_SOURCE = "pattern_source"
    PATTERN_REALM = "pattern_realm"
    RUBBING_ALIGN = "rubbing_align"
    POINT_TRACE = "point_trace"
    REPAIR_MINIGAME = "repair_minigame"
    ENDING = "ending"
```

### 7.2 状态切换规则

基本流转：

```text
MENU
→ INTRO
→ PLAYING
→ DIALOGUE / INVENTORY / CODEX / PATTERN_REALM / REPAIR_MINIGAME
→ PLAYING
→ ENDING
```

规则：

1. 非 `PLAYING` 状态下，玩家不能继续移动；
2. 进入背包、图鉴、对话、修复小游戏时应暂停地图交互；
3. 从纹样空间返回时，应恢复原场景、原任务和背包状态；
4. 状态切换不能清空任务、背包、图鉴、银光值。

## 八、地图与 Tiled 规范

### 8.1 地图规格

统一规格：

```text
视窗：800 × 600
瓦片：32 × 32
地图尺寸：必须尽量为 32 的整数倍
```

推荐地图：

| 文件 | 用途 |
|---|---|
| `village.tmx` | 村寨入口，新手引导 |
| `workshop.tmx` | 银饰工坊，敲银修复 |
| `market.tmx` | 山地集市，材料与 NPC 线索 |
| `mountain.tmx` | 自然场景寻源 |
| `festival.tmx` | 节庆展示与结局 |
| `pattern_mountain.tmx` | 山纹路径小关卡 |
| `pattern_water.tmx` | 水波纹路径小关卡，可选 |
| `pattern_sun.tmx` | 太阳纹 / 火纹小关卡，可选 |

### 8.2 图层命名规范

Tiled 图层必须使用英文命名。

标准图层：

```text
background
ground
decoration
collision
object_points
npc_points
pattern_points
clue_points
checkpoint_points
trigger_points
foreground
```

不得使用：

```text
图层1
对象
NPC点
碰撞层1
```

### 8.3 对象层字段规范

所有对象必须至少包含：

```text
id
type
```

#### NPC 对象

```text
id: elder
name: 村寨长者
type: npc
dialogue_id: elder_dialogue
task_id: main_01
```

#### 道具对象

```text
id: silver_fragment_01
name: 银饰碎片
type: item
item_type: fragment
codex_unlock: silver_earring
```

#### 纹样寻源点

```text
id: pattern_mountain
name: 山纹寻源点
type: pattern_source
pattern_id: mountain_pattern
challenge_type: path_maze
reward_item: mountain_pattern_rubbing
target_scene: pattern_mountain
codex_unlock: codex_mountain_pattern
required_task: main_04
```

#### 自然线索点

```text
id: water_clue_01
name: 河边水流
type: clue
pattern_id: water_pattern
clue_group: water_source
required_count: 3
```

#### 纹样检查点

```text
id: mountain_checkpoint_01
type: pattern_checkpoint
pattern_id: mountain_pattern
order: 1
next_checkpoint: mountain_checkpoint_02
```

#### 场景出口

```text
id: exit_to_workshop
type: scene_exit
target_scene: workshop
target_spawn: workshop_entry
```

### 8.4 地图读取规范

程序读取对象层时：

1. 必须检查 `id` 和 `type`；
2. 缺少必要字段时输出调试提示；
3. 不应因为一个非关键对象错误导致整个游戏崩溃；
4. 坐标统一使用世界坐标；
5. 绘制时再转换为屏幕坐标。

## 十一、核心模块开发规范

### 11.1 地图模块

必须实现：

1. 加载 TMX；
2. 绘制可见瓦片；
3. 读取碰撞层；
4. 读取对象层；
5. 支持场景切换；
6. 支持纹样空间地图。

禁止：

1. 每帧绘制整张超大地图；
2. 硬编码所有 NPC 坐标；
3. 使用中文图层名；
4. 忽略对象层属性检查。

### 11.2 玩家模块

必须实现：

1. WASD / 方向键移动；
2. 碰撞检测；
3. E 键交互；
4. 朝向或基础动画；
5. 在主地图和纹样空间共用移动逻辑。

### 11.3 任务模块

任务状态统一为：

```text
not_started
in_progress
ready_to_submit
completed
```

任务推进必须由 `TaskManager` 统一处理，不得在 NPC、Item、RepairMiniGame 中分散修改大量任务逻辑。

### 11.4 纹样寻源模块

必须优先实现：

```text
source_hunt
path_maze
```

可选实现：

```text
align_rubbing
point_trace
```

完成纹样寻源后必须更新：

1. 背包；
2. 图鉴；
3. 任务；
4. 已完成纹样状态；
5. 反馈提示。

### 11.5 敲银修复模块

敲银修复必须和拓印区分：

```text
拓印 / 寻源：获得纹样拓片
敲银：使用拓片和材料修复银饰
```

修复成功后必须更新：

1. 银饰部件状态；
2. 套装进度；
3. 银光值；
4. 图鉴；
5. 任务状态。

### 11.6 图鉴模块

图鉴分为：

1. 银饰图鉴；
2. 纹样图鉴。

图鉴文案必须简短、准确、尊重，避免未经证实的绝对化说法。

### 11.7 银光值模块

银光值由 `ScoreManager` 管理。

推荐规则：

```text
修复银饰部件：+10
获得清晰拓片：+5
完成支线任务：+10
找到隐藏银纹：+5
敲银失败：-3
普通错误选择：-2
```

银光值不应影响主线是否能完成，只影响最终评价。

## 十二、AI / Codex 协作规范

### 12.1 给 Codex 的任务必须明确

每次只让 Codex 做一个阶段。

推荐提示词格式：

```text
请只实现【地图与摄像机模块】。
要求：
1. 使用 Pygame + PyTMX；
2. 读取 maps/village.tmx；
3. 实现 Camera 类；
4. 视窗 800×600；
5. 地图大于视窗；
6. 不要实现 NPC、任务、背包；
7. 修改后说明改了哪些文件、如何运行、如何测试。
```

不推荐：

```text
请直接把整个游戏做完。
```

### 12.2 Codex 不允许做的事

Codex 不得：

1. 随意加入战斗系统；
2. 随意加入联网功能；
3. 随意改项目目录；
4. 把所有功能写进 main.py；
5. 删除已有文档；
6. 删除已有资源；
7. 修改与当前任务无关的大量文件；
8. 生成不可运行的大段伪代码后不说明。

### 12.3 Codex 输出必须包含

每次让 Codex 修改后，要求它输出：

1. 修改了哪些文件；
2. 新增了哪些类；
3. 新增了哪些函数；
4. 如何运行；
5. 如何测试；
6. 当前未完成内容；
7. 是否影响已有功能。

### 12.4 推荐实现阶段

```text
阶段 1：基础工程与窗口
阶段 2：地图加载与摄像机
阶段 3：玩家移动与碰撞
```

## 十五、异常处理规范

### 15.1 地图异常

地图文件缺失时：

1. 输出错误路径；
2. 返回主菜单或加载测试地图；
3. 不应直接无提示崩溃。

### 15.2 资源异常

图片缺失时使用占位图。  
音效缺失时跳过播放。  
JSON 缺失时给出错误提示。

### 15.3 对象层异常

对象缺少 `id` 或 `type` 时：

1. 输出调试提示；
2. 跳过该对象；
3. 不影响其他对象加载。

### 15.4 任务异常

任务状态不存在或前置条件错误时：

1. HUD 给出温和提示；
2. 控制台输出调试信息；
3. 不让玩家进入死循环；
4. 关键道具不永久消失。

## 十八、最终说明

本开发规范用于约束《银纹秘境：彝饰守护者》的实际代码开发过程。

本项目的技术重点不是功能数量无限增加，而是通过清晰的工程结构完成一个可运行、可测试、可展示的 2D Pygame 游戏 Demo。

开发过程中应始终坚持：

```text
文档先行
模块拆分
数据驱动
主线闭环
逐步实现
每步可测
```

最终目标是让 Codex 能根据 GDD、SRS、FSD、结构化地图设计和本开发规范，分阶段生成稳定代码，而不是一次性生成难以维护的大段混乱代码。
