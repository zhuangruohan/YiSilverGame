# 《银纹秘境：彝饰守护者》角色动画帧命名规范 + 每个角色需要的帧表

## 当前实现同步说明（2026-07-01）

当前角色与 NPC 图片资源位于 `assets/sprites/`。本文档用于规范后续扩展素材命名，不代表当前所有动画状态均已完整实现。

当前演示主线为：

```text
村寨长者对话 -> 集市寻源 -> 河谷影纹净化 -> 节庆广场守护 -> Ending
```

## 1. 文档用途

本文件用于统一项目中角色动画帧的命名方式、文件夹结构、导出规则与最小需求清单，方便：

1. 使用 GPT / 图像工具继续生成角色帧；
2. 后续在 Pygame 中按统一规则加载精灵帧；
3. 与 Tiled / JSON / 代码逻辑对接；
4. 避免后期出现素材混乱、重名、方向不统一的问题。

---

## 2. 总体命名规范

### 2.1 文件命名总格式

统一采用：

```text
[角色ID]_[动作类型]_[方向或状态]_[序号].png
```

例如：

```text
player_walk_down_01.png
player_walk_down_02.png
player_walk_down_03.png
elder_talk_front_01.png
smith_hammer_front_02.png
host_introduce_front_01.png
```

---

## 3. 命名字段说明

### 3.1 角色 ID（character_id）

建议统一如下：

| 角色 | 角色 ID | 说明 |
|---|---|---|
| 主角 | `player` | 玩家操作角色 |
| 村寨长者 | `elder` | 新手引导 / 主线发布 |
| 银匠师傅 | `smith` | 工坊修复教学 |
| 集市阿妈 | `market_auntie` | 集市线索 NPC |
| 节庆主持人（小女孩） | `host` | 节庆展示 / 结局主持 |

| 村民男 | `villager_male_01` | 普通路人 |
| 村民女 | `villager_female_01` | 普通路人 |

如果后面继续加 NPC，可以按：

```text
villager_male_02
villager_female_02

```

---

### 3.2 动作类型（action）

建议统一使用这些英文关键词：

| 动作类型 | 关键词 | 说明 |
|---|---|---|
| 静止 | `idle` | 站立不动 |
| 行走 | `walk` | 角色移动 |
| 跑步（可选） | `run` | 如果后面加速移动 |
| 阅读 | `read` | 看手册 / 看纸张 |
| 拾取 | `pick` | 弯腰拾取物品 |
| 蹲下 | `kneel` | 蹲下观察 / 拓印前动作 |
| 拿袋子 | `carry` | 拿着材料袋 |
| 修复准备 | `repair_ready` | 准备敲银 |
| 敲银 | `hammer` | 敲银修复过程 |
| 展示成果 | `show` | 举起银饰 |
| 成功庆祝 | `celebrate` | 修复成功 / 开心 |
| 说话 | `talk` | 对话中动作 |
| 指向 | `point` | 指引方向 |
| 介绍 | `introduce` | 主持人介绍 |
| 解释 | `explain` | NPC 讲解 |
| 微笑 | `smile` | 表情变化 |
| 请求帮助 | `ask_help` | 小朋友支线 |
| 开心 | `happy` | 支线完成 |

---

### 3.3 方向或状态（direction/state）

#### 用于主角移动的方向字段：

| 方向 | 关键词 |
|---|---|
| 上 | `up` |
| 下 | `down` |
| 左 | `left` |
| 右 | `right` |
| 正面 | `front` |
| 背面 | `back` |
| 左侧 | `side_left` |
| 右侧 | `side_right` |

说明：
- **主角移动动画**建议优先使用 `up / down / left / right`。
- **NPC 站立/讲话动作**建议优先使用 `front`。
- 如果某些 NPC 是明显侧身图，可以使用 `side_left` 或 `side_right`。

---

### 3.4 序号（frame index）

统一使用两位数：

```text
01
02
03
04
```

不要写成：

```text
1, 2, 3
```

这样后面程序排序更稳定。

## 13. 导出与整理规范

### 13.1 图片格式
- 统一使用 `.png`
- 建议透明背景
- 单帧尺寸尽量统一

### 13.2 建议尺寸
- 主角/NPC：`48x64` 或 `64x64`
- 站立型 NPC 建议统一同一高度
- 后期放入游戏前，保证脚底位置基本一致

### 13.3 导出方式
你可以选择两种：

#### 方式 A：单帧文件
优点：最清楚，适合 Pygame 逐帧加载。

#### 方式 B：spritesheet
例如：

```text
player_walk_down_sheet.png
smith_hammer_sheet.png
```

但如果你现在还在不断改图，建议先用 **单帧文件**。

## 14. 与代码对接建议

Pygame 中建议这样组织：

```python
animations = {
    "idle_down": ["player_idle_down_01.png"],
    "walk_down": [
        "player_walk_down_01.png",
        "player_walk_down_02.png",
        "player_walk_down_03.png",
    ],
    "hammer_front": [
        "player_hammer_front_01.png",
        "player_hammer_front_02.png",
        "player_hammer_front_03.png",
    ]
}
```

这样你后面改帧数也方便。
