# 《银纹秘境：彝饰守护者》功能规格说明书 FSD（纹样寻源优化版）

## 1. 文档说明

### 1.1 文档目的

本文档用于细化《银纹秘境：彝饰守护者》的具体功能规格，说明每个功能模块的功能目标、触发方式、输入输出、处理流程、状态变化、异常处理和验收标准。

本版本根据新版游戏策划与新版 SRS 进行同步调整，重点将旧版“纹样拓印”升级为“纹样寻源系统”，将旧版“修复一件银饰”升级为“修复一套节庆银饰”，并加入纹样空间小关卡、自然场景寻源、走纹样迷宫、敲银修复、银饰套装进度、银光值和结局评价。

本文档是游戏策划文档 GDD 与软件需求规格说明书 SRS 之间的衔接文档：

1. GDD 说明游戏设计目标、玩法体验和文化表达；
2. SRS 说明系统需求、接口需求、非功能需求和验收标准；
3. FSD 进一步说明每个功能如何具体实现和表现，供开发、测试和 AI/Codex 协作实现使用。

### 1.2 适用对象

本文档适用于：

1. 项目开发者；
2. AI/Codex 协作开发工具；
3. 测试人员；
4. 课程验收人员；
5. 后续维护人员。

### 1.3 功能规格原则

本项目功能设计遵循以下原则：

1. 优先保证完整主线流程；
2. 功能围绕彝族银饰文化展开；
3. 不加入复杂战斗、商店、联网和大型开放世界；
4. 地图、角色、NPC、道具、任务、纹样寻源、修复、图鉴、UI 等功能应模块化；
5. 所有核心功能应能被测试和验收；
6. 缺少美术资源时允许使用占位图、简单线条、色块或基础几何图形代替；
7. AI 协作开发时每次只实现一个功能模块，避免一次性生成过多不可控代码；
8. 纹样寻源优先实现 source_hunt 自然场景寻源和 path_maze 走纹样迷宫；align_rubbing 纹样对齐和 point_trace 关键点描摹作为优秀扩展；
9. 文化说明使用“游戏化转译”表达，不将缺少明确出处的纹样寓意写成绝对民俗结论。

P2：优秀扩展。完成后项目展示效果、体验和规范性更强。

## 3. 全局功能规则

### 3.1 游戏运行状态

系统应至少包含以下游戏状态：

| 状态名 | 说明 |
|---|---|
| MENU | 主菜单状态。 |
| INTRO | 剧情引导状态。 |
| PLAYING | 主地图游玩状态。 |
| DIALOGUE | NPC 对话状态。 |
| INVENTORY | 背包界面状态。 |
| CODEX | 图鉴界面状态。 |
| PATTERN_SOURCE | 纹样寻源说明或入口状态，可选。 |
| PATTERN_REALM | 纹样空间小关卡状态。 |
| SOURCE_HUNT | 自然场景寻源状态，可与 PLAYING 合并实现。 |
| RUBBING_ALIGN | 纹样对齐状态，可选。 |
| POINT_TRACE | 关键点描摹状态，可选。 |
| REPAIR_MENU | 工坊修复选择界面状态。 |
| REPAIR_MINIGAME | 敲银修复小游戏状态。 |
| FESTIVAL_SHOW | 节庆展示状态。 |
| ENDING | 结局评价状态。 |
| DEBUG | 调试叠加状态，可选。 |

### 3.2 状态切换规则

基本状态流转如下：

```text
MENU
  ↓
INTRO
  ↓
PLAYING
  ↓
DIALOGUE / INVENTORY / CODEX
  ↓
PLAYING
  ↓
PATTERN_SOURCE / SOURCE_HUNT / PATTERN_REALM / RUBBING_ALIGN / POINT_TRACE
  ↓
PLAYING
  ↓
REPAIR_MENU
  ↓
REPAIR_MINIGAME
  ↓
PLAYING
  ↓
FESTIVAL_SHOW
  ↓
ENDING
```

状态切换要求：

1. 主菜单点击“开始游戏”后进入 INTRO 或 PLAYING；
2. 玩家靠近 NPC 并按 E 后进入 DIALOGUE；
3. 玩家按 I 后进入 INVENTORY；
4. 玩家按 G 后进入 CODEX；
5. 玩家与纹样寻源点交互后，根据 challenge_type 进入对应纹样玩法；
6. 玩家与修复台交互且满足条件后进入 REPAIR_MENU 或 REPAIR_MINIGAME；
7. 完成最终展示后进入 ENDING；
8. 非 PLAYING 状态下，玩家角色不应继续在主地图移动；
9. 返回 PLAYING 状态后，任务、背包、图鉴、银光值、已完成纹样状态应保持；
10. 纹样空间完成或退出后，应能回到进入前的场景和坐标附近。

### 3.3 交互优先级规则

当玩家附近存在多个可交互对象时，交互优先级为：

1. NPC；
2. 任务触发点；
3. 修复台或展示台；
4. 纹样寻源点；
5. 自然线索点；
6. 可拾取物品；
7. 场景出口；
8. 普通提示点。

如果多个对象距离相近，系统应选择距离玩家最近的对象。

### 3.4 坐标规则

系统中应区分：

1. 世界坐标：对象在完整地图中的位置；
2. 屏幕坐标：对象显示在视窗中的位置；
3. 摄像机偏移：当前视窗相对于完整地图的偏移量。

坐标转换规则：

```text
screen_x = world_x - camera_x
screen_y = world_y - camera_y
```

所有交互、碰撞、任务判断应优先使用世界坐标。

### 3.5 数据状态规则

系统至少需要维护以下全局数据：

1. 当前场景 current_scene；
2. 玩家位置 player_position；
3. 当前任务 active_task；
4. 已完成任务 completed_tasks；
5. 背包 inventory；
6. 已完成纹样 completed_patterns；
7. 已获得拓片 pattern_rubbings；
8. 已收集线索 collected_clues；
9. 已修复银饰部件 repaired_parts；
10. 图鉴解锁状态 unlocked_codex；
11. 银光值 silver_score；
12. 支线完成状态 completed_side_tasks。

## F-01 游戏启动与主菜单

### 4.1 功能目标

该模块用于启动游戏、显示游戏标题、进入游戏流程或退出程序。

### 4.2 涉及界面

1. 主菜单界面；
2. 操作说明界面，可选；
3. 退出确认提示，可选。

### 4.3 输入

1. 鼠标点击开始按钮；
2. 鼠标点击退出按钮；
3. Enter 确认，可选；
4. Esc 退出，可选。

### 4.4 输出

1. 显示游戏标题《银纹秘境：彝饰守护者》；
2. 显示开始游戏按钮；
3. 显示退出游戏按钮；
4. 点击开始后切换到剧情引导或村寨入口；
5. 点击退出后关闭程序。

### 4.5 主流程

1. 玩家运行 main.py；
2. 系统初始化 Pygame；
3. 系统创建 800 × 600 游戏窗口；
4. 系统加载主菜单背景、标题和按钮；
5. 玩家点击“开始游戏”；
6. 系统初始化 GameState、TaskManager、Inventory、Codex、ScoreManager；
7. 系统切换到 INTRO 或 PLAYING 状态；
8. 游戏正式开始。

### 4.6 异常处理

1. 主菜单背景缺失时使用纯色背景；
2. 字体缺失时使用默认字体；
3. 按钮图片缺失时使用矩形按钮；
4. 初始化失败时在控制台输出错误信息。

### 4.7 验收标准

1. 运行程序后能看到主菜单；
2. 点击开始游戏能进入游戏；
3. 点击退出游戏能关闭程序；
4. 主菜单不会卡死；
5. 缺少非关键素材时不应导致程序崩溃。

## F-02 游戏状态管理

### 5.1 功能目标

该模块用于统一管理游戏所处状态，避免主地图、对话、背包、图鉴、纹样小关卡、修复小游戏和结局界面之间逻辑混乱。

### 5.2 状态列表

必须支持：MENU、INTRO、PLAYING、DIALOGUE、INVENTORY、CODEX、PATTERN_REALM、REPAIR_MENU、REPAIR_MINIGAME、FESTIVAL_SHOW、ENDING。

可选支持：SOURCE_HUNT、RUBBING_ALIGN、POINT_TRACE、DEBUG。

### 5.3 主流程

1. 游戏启动后进入 MENU；
2. 选择开始后进入 INTRO；
3. 剧情引导结束后进入 PLAYING；
4. 玩家触发不同功能时切换对应状态；
5. 功能界面关闭后返回 PLAYING；
6. 完成纹样空间后返回进入前场景；
7. 完成敲银修复后返回工坊场景；
8. 完成最终展示后进入 ENDING。

### 5.4 状态限制

1. 在 DIALOGUE 状态下，玩家不可移动；
2. 在 INVENTORY 状态下，玩家不可移动；
3. 在 CODEX 状态下，玩家不可移动；
4. 在 PATTERN_REALM 状态下，只更新纹样空间地图，不更新主地图 NPC 与物品；
5. 在 REPAIR_MENU 和 REPAIR_MINIGAME 状态下，地图暂停更新；
6. 在 ENDING 状态下，只允许重新开始或退出。

### 5.5 验收标准

1. 各状态能正常进入和退出；
2. 状态切换后不会丢失任务、背包、图鉴、银光值和已完成纹样数据；
3. 非主游戏状态下玩家不会继续移动；
4. 状态切换不会导致黑屏或卡死；
5. 纹样空间完成后能返回原场景。

## F-03 地图与场景系统

### 6.1 功能目标

该模块用于加载 Tiled 地图文件，绘制地图图层，管理不同场景之间的切换，并支持纹样空间小关卡地图。

### 6.2 场景列表

主要场景：

1. village：村寨入口；
2. workshop：银饰工坊；
3. market：山地集市；
4. mountain：山地与自然场景；
5. festival：节庆广场。

纹样空间场景，可选但建议至少实现一个：

1. pattern_mountain：山纹路径空间；
2. pattern_water：水波纹路径空间；
3. pattern_sun：太阳纹或火纹空间。

### 6.3 地图规格

1. 瓦片大小统一建议为 32 × 32 像素；
2. 主地图应大于视窗，体现滚动探索；
3. 纹样空间地图可以小于主地图，但需要支持碰撞、检查点和完成区域；
4. 地图尺寸应尽量为 32 的整数倍。

### 6.4 图层规格

每张主地图推荐包含：

1. background；
2. ground；
3. decoration；
4. collision；
5. object_points；
6. npc_points；
7. pattern_points；
8. clue_points；
9. trigger_points；
10. foreground。

纹样空间地图推荐包含：

1. background；
2. pattern_path；
3. collision；
4. checkpoint_points；
5. trigger_points；
6. foreground。

### 6.5 主流程

1. SceneManager 请求加载场景；
2. TileMap 读取对应 TMX 文件；
3. 系统读取地图宽高、瓦片大小、图层信息；
4. 系统加载地面层、装饰层、前景层；
5. 系统读取碰撞层和对象层；
6. 系统生成 NPC、物品、触发区、纹样寻源点、自然线索点；
7. 若为纹样空间地图，系统生成检查点和终点；
8. 系统将玩家放置到指定出生点；
9. 场景进入对应状态。

### 6.6 场景切换流程

1. 玩家进入 trigger_points 中的 scene_exit 区域；
2. 系统读取 target_scene；
3. 系统读取 target_spawn；
4. 系统保存当前任务、背包、图鉴、银光值和已完成纹样状态；
5. 系统卸载当前场景；
6. 系统加载目标场景；
7. 玩家出现在目标出生点。

### 6.7 纹样空间进入与返回流程

1. 玩家与 PatternSourcePoint 交互；
2. 系统读取 pattern_id、challenge_type、target_scene、reward_item；
3. 系统记录 return_scene 和 return_position；
4. 系统加载 target_scene；
5. 玩家进入 PATTERN_REALM 状态；
6. 完成纹样后获得拓片；
7. 系统返回 return_scene，并恢复玩家位置；
8. 系统更新任务、背包和图鉴。

### 6.8 异常处理

1. 地图文件缺失时显示错误提示；
2. 纹样空间 target_scene 缺失时不进入关卡，并提示“纹样空间暂未开放”；
3. 地图对象属性缺失时使用默认值或在调试信息中提示；
4. 地图加载失败时返回安全场景或主菜单。

### 6.9 验收标准

1. 主要场景能被加载；
2. 至少一个纹样空间能被加载；
3. 地图图层显示顺序正确；
4. 玩家能在场景之间切换；
5. 切换场景后任务和背包数据不丢失；
6. 地图大于视窗时能正常显示子区域；
7. 纹样空间完成后能正常返回。

## F-04 摄像机与视窗系统

### 7.1 功能目标

实现地图大于视窗时的摄像机跟随效果，让玩家在大地图或纹样空间地图中移动时，视窗跟随角色显示当前位置。

### 7.2 输入

1. 玩家世界坐标；
2. 当前地图宽度；
3. 当前地图高度；
4. 视窗宽度；
5. 视窗高度。

### 7.3 输出

1. camera_x；
2. camera_y；
3. 当前屏幕显示区域；
4. 对象屏幕坐标。

### 7.4 摄像机跟随规则

1. 玩家位于地图中部时，摄像机尽量让玩家居中；
2. 玩家靠近地图边缘时，摄像机停止移动；
3. 摄像机不能显示地图外区域；
4. 所有地图对象绘制时都需要减去摄像机偏移；
5. 纹样空间地图如果小于视窗，可以固定居中显示；如果大于视窗，使用同样的摄像机规则。

### 7.5 计算规则

```text
camera_x = player_x - viewport_width / 2
camera_y = player_y - viewport_height / 2
```

边界限制：

```text
0 ≤ camera_x ≤ map_width - viewport_width
0 ≤ camera_y ≤ map_height - viewport_height
```

### 7.6 异常处理

1. 如果地图宽度小于视窗宽度，camera_x 固定为 0 或居中偏移；
2. 如果地图高度小于视窗高度，camera_y 固定为 0 或居中偏移；
3. 坐标计算出现异常时，输出调试信息。

### 7.7 验收标准

1. 玩家移动时摄像机跟随；
2. 摄像机不会越界；
3. 屏幕不会显示地图外空白区域；
4. NPC、物品、玩家、纹样点绘制位置正确；
5. 交互范围不受摄像机偏移影响。

## F-05 玩家角色系统

### 8.1 功能目标

实现玩家角色的创建、移动、动画、交互和状态管理。玩家在普通地图中表现为“银纹守护者”，在纹样空间中可使用同一角色，也可简化为“银光点”。

### 8.2 玩家属性

| 属性 | 说明 |
|---|---|
| player_id | 玩家标识。 |

### 8.3 输入

1. WASD 或方向键；
2. E 交互键；
3. I 背包键；
4. G 图鉴键；
5. 空格敲银确认；
6. Esc 返回键。

### 8.4 输出

1. 玩家坐标变化；
2. 玩家朝向变化；
3. 玩家动画变化；
4. 触发交互事件；
5. 触发界面切换；
6. 在纹样空间中触发检查点。

### 8.5 移动流程

1. 系统读取键盘方向输入；
2. 根据输入计算新位置；
3. 判断新位置是否越界；
4. 判断新位置是否碰撞；
5. 如果合法，更新玩家世界坐标；
6. 更新摄像机；
7. 绘制玩家到屏幕坐标。

### 8.6 交互流程

1. 玩家按 E；
2. 系统检测交互范围内对象；
3. 按交互优先级选择对象；
4. 触发 NPC 对话、拾取、纹样寻源、线索记录、修复台或场景出口；
5. 根据触发结果更新任务、背包、图鉴、银光值或状态。

### 8.7 纹样空间移动规则

1. 使用与主地图相同的移动键位；
2. 玩家必须沿可通行路径移动；
3. 玩家经过检查点时触发点亮；
4. 玩家走错路径时可显示提示，但不 Game Over；
5. 玩家按 Esc 可退出纹样空间，可选。

### 8.8 验收标准

1. 玩家可以上下左右移动；
2. 玩家不能穿越障碍物；
3. 玩家不能走出地图；
4. 玩家靠近可交互对象时能触发交互；
5. 玩家动画或朝向能反映移动方向；
6. 玩家能在纹样空间中移动并触发检查点。

## F-06 碰撞检测系统

### 9.1 功能目标

防止玩家穿越建筑物、树木、摊位、墙壁、地图边界和纹样空间路径边界。

### 9.2 碰撞对象来源

1. Tiled collision 层；
2. 地图边界；
3. 特殊不可通行对象；
4. 纹样空间路径边界；
5. 可选：动态对象碰撞。

### 9.3 碰撞流程

1. 玩家输入移动方向；
2. 系统生成预测位置 next_rect；
3. 系统判断 next_rect 是否超出地图边界；
4. 系统判断 next_rect 是否与 collision_rects 重叠；
5. 如果发生碰撞，取消移动；
6. 如果无碰撞，更新玩家位置。

### 9.4 碰撞规则

1. 碰撞检测使用世界坐标；
2. 绘制使用屏幕坐标；
3. 碰撞区域可以比美术图略小，避免视觉卡顿；
4. 玩家碰撞框可小于角色图片，提升移动体验；
5. 纹样空间中碰撞区域应引导玩家沿纹样路径移动。

### 9.5 验收标准

1. 玩家不能穿过房屋；
2. 玩家不能穿过树木；
3. 玩家不能穿过摊位；
4. 玩家不能走出地图边界；
5. 玩家在纹样空间中不能穿过路径边界；
6. 玩家不会卡入障碍物内部。

## F-11 纹样寻源系统

### 14.1 功能目标

纹样寻源系统是新版项目的核心特色玩法，用于替代旧版“靠近纹样点按 E 直接获得拓片”的简单拓印。玩家需要通过自然场景寻源、走纹样迷宫、纹样对齐或关键点描摹等方式理解纹样来源，并获得对应纹样拓片。

### 14.2 纹样寻源类型

| challenge_type | 名称 | 优先级 | 说明 |
|---|---|---|---|
| source_hunt | 自然场景寻源 | P1 | 收集自然或生活线索后获得拓片。 |
| path_maze | 走纹样迷宫 | P1 | 在纹样空间沿路径经过检查点。 |
| align_rubbing | 纹样对齐 | P2 | 移动拓印纸使其与底层纹样重合。 |
| point_trace | 关键点描摹 | P2 | 用鼠标依次点击或拖过关键点。 |

### 14.3 触发条件

1. 玩家处于 PLAYING 状态；
2. 玩家靠近 pattern_source 对象；
3. 当前任务允许进入纹样寻源，或该纹样已开放；
4. 该纹样未完成，或允许重复练习但不重复发放奖励；
5. 玩家按 E。

### 14.4 输入

1. E：进入纹样寻源点；
2. WASD / 方向键：纹样空间移动或对齐操作；
3. 鼠标：可选的对齐、点击或描摹；
4. Esc：退出当前纹样玩法，可选。

### 14.5 输出

1. 进入对应纹样玩法；
2. 显示纹样名称、任务目标和进度；
3. 完成后获得 reward_item；
4. 背包新增纹样拓片；
5. 图鉴解锁对应纹样条目；
6. TaskManager 推进任务；
7. ScoreManager 根据表现调整银光值；
8. PatternSourcePoint 状态改为 completed。

### 14.6 通用流程

1. 玩家靠近纹样寻源点；
2. 系统显示“按 E 进入纹样寻源”；
3. 玩家按 E；
4. 系统读取 pattern_id、challenge_type、reward_item、required_task；
5. 系统检查任务条件；
6. 根据 challenge_type 调用对应玩法；
7. 玩家完成玩法目标；
8. 系统发放拓片；
9. 系统播放获得拓片反馈；
10. 系统更新背包、任务、图鉴、银光值；
11. 返回 PLAYING。

### 14.7 异常处理

1. required_task 未满足时提示“现在还没有理解这个纹样的线索”；
2. challenge_type 未识别时提示配置错误，并不进入关卡；
3. reward_item 缺失时提示调试错误，不发放空道具；
4. 玩家已完成该纹样时提示“这个纹样已经获得拓片”；
5. 纹样玩法失败时允许重新尝试。

### 14.8 验收标准

1. 玩家能与纹样寻源点交互；
2. 系统能根据 challenge_type 进入对应玩法；
3. 至少实现 source_hunt 或 path_maze 一种玩法；
4. 标准版本至少实现两种纹样寻源玩法；
5. 完成后背包获得对应拓片；
6. 拓片能用于工坊修复；
7. 重复完成不会重复发放奖励；
8. 纹样寻源完成后任务与图鉴正确更新。

## F-12 走纹样迷宫系统

### 15.1 功能目标

走纹样迷宫用于表现“玩家亲自走出纹样”。系统加载一张纹样空间小地图，玩家沿着形似纹样的路径移动，按顺序经过检查点，最终让路径发光并生成拓片。

### 15.2 适用纹样

1. 山纹：折线山脊路线；
2. 水波纹：波浪形路线；
3. 火纹 / 太阳纹：中心向外扩散路线；
4. 花鸟纹：可选，适合高级路径或点线组合。

### 15.3 输入

1. WASD / 方向键：控制角色或银光点移动；
2. E：确认或进入终点，可选；
3. Esc：退出纹样空间，可选。

### 15.4 输出

1. 当前检查点进度；
2. 已点亮路径；
3. 提示文本；
4. 完成后的纹样合成动画；
5. 纹样拓片奖励。

### 15.5 对象层配置

检查点对象示例：

```text
id: mountain_checkpoint_01
type: pattern_checkpoint
pattern_id: mountain_pattern
order: 1
next_checkpoint: mountain_checkpoint_02
```

终点对象示例：

```text
id: mountain_finish
type: pattern_finish
pattern_id: mountain_pattern
required_last_checkpoint: mountain_checkpoint_05
```

### 15.6 主流程

1. 系统加载 pattern_mountain.tmx 或对应纹样空间地图；
2. 系统生成玩家或银光点；
3. 系统读取 checkpoint_points 并按 order 排序；
4. 当前目标检查点设为 order=1；
5. 玩家移动到当前检查点范围；
6. 检查点点亮，当前目标变为下一个；
7. 玩家按顺序完成全部检查点；
8. 玩家进入终点；
9. 系统播放完整纹样合成反馈；
10. 系统发放对应拓片；
11. 系统返回原场景。

### 15.7 错误与失败处理

1. 玩家进入非当前检查点时提示“请沿着发光线条继续前进”；
2. 玩家偏离路径时通过碰撞或提示引导回路径；
3. 玩家中途退出时不发放拓片；
4. 玩家可重新进入挑战；
5. 失败不扣除关键道具，不 Game Over。

### 15.8 评分规则，可选

1. 按顺序完成且未走错：获得 clear 拓片；
2. 多次走错但完成：获得 normal 拓片；
3. clear 拓片可提供少量银光值加成。

### 15.9 验收标准

1. 纹样空间地图能加载；
2. 玩家能在纹样空间中移动；
3. 检查点能按顺序触发；
4. 检查点触发后有点亮反馈；
5. 全部检查点完成后获得拓片；
6. 完成后能返回原场景；
7. 重复挑战不会重复发放奖励。

## F-13 自然场景寻源系统

### 16.1 功能目标

自然场景寻源用于表现“纹样来自自然或生活观察”。玩家在山地、河边、集市、火塘等场景中收集 2 到 3 个线索，理解纹样来源后获得对应拓片。

### 16.2 适用纹样

| 纹样 | 场景 | 线索示例 |
|---|---|---|
| 水波纹 | 河边 / 银器清洗处 | 水流、清洗处、布料波纹。 |
| 山纹 | 山地 / 石阶 | 远山、山路石阶、山脊线。 |
| 火纹 / 太阳纹 | 火塘 / 节庆灯火 | 火光、日光、银饰反光。 |
| 花鸟纹 | 集市 / 服饰区 | 花、鸟、服饰图案。 |

### 16.3 输入

1. 玩家移动到线索点；
2. E：记录线索；
3. G：查看图鉴线索，可选。

### 16.4 输出

1. 已记录线索数量；
2. 线索记录提示；
3. 达到 required_count 后获得拓片；
4. 图鉴线索解锁；
5. 背包更新。

### 16.5 对象层配置

```text
id: water_clue_01
name: 河边水流
type: clue
pattern_id: water_pattern
clue_group: water_source
required_count: 3
description: 水流的曲线让你想到银饰上的波纹。
```

### 16.6 主流程

1. 玩家根据 NPC 提示进入自然或生活场景；
2. 场景中显示若干 clue 对象；
3. 玩家靠近 clue 对象；
4. 系统显示“按 E 记录线索”；
5. 玩家按 E；
6. 系统记录 clue_id；
7. 系统统计 clue_group 完成数量；
8. 若数量达到 required_count，则触发纹样理解完成；
9. 系统显示“你理解了水波纹的来源”；
10. 系统发放 water_pattern_rubbing；
11. 系统更新任务、背包、图鉴和银光值。

### 16.7 异常处理

1. 已记录线索再次交互时提示“这条线索已经记录过”；
2. clue_group 配置缺失时不计入完成，输出调试信息；
3. required_count 缺失时默认 required_count=1；
4. 玩家未接任务时可提示“现在还不需要记录这个线索”。

### 16.8 验收标准

1. 线索点能显示在场景中；
2. 玩家能记录线索；
3. 已记录线索不会重复计数；
4. 达到 required_count 后获得拓片；
5. 获得拓片后能用于工坊修复；
6. 完成后图鉴与任务正确更新。

## F-14 纹样对齐与关键点描摹系统（可选扩展）

### 17.1 功能目标

该模块用于增强拓印表现。纹样对齐模拟“拓印纸与底层纹样重合”，关键点描摹模拟“线条生成”。两者均为 P2 优秀扩展，不作为最低可运行版本必需功能。

### 17.2 纹样对齐流程

1. 系统进入 RUBBING_ALIGN 状态；
2. 界面显示底层浅色纹样；
3. 界面显示一张半透明拓印纸；
4. 玩家使用方向键或鼠标调整拓印纸位置；
5. 系统计算中心点距离、角度偏差或模拟重合度；
6. 重合度达到阈值后成功；
7. 系统根据重合度发放 normal 或 clear 拓片。

### 17.3 纹样对齐判定

建议判定：

```text
重合度 ≥ 80%：拓印成功，获得普通拓片。
重合度 ≥ 95%：拓印优秀，获得清晰拓片，银光值加成。
```

实际实现可使用：

1. 中心点距离；
2. 旋转角度差；
3. 预设判定框；
4. 简化数值模拟。

不需要实现真实图像识别。

### 17.4 关键点描摹流程

1. 系统进入 POINT_TRACE 状态；
2. 屏幕显示若干发光关键点；
3. 玩家用鼠标按顺序点击或拖过关键点；
4. 系统判断顺序和距离；
5. 正确则点亮当前线段；
6. 全部点完成后生成完整纹样；
7. 系统发放拓片。

### 17.5 异常处理

1. 玩家点击错误点时提示“请按纹样线索顺序描摹”；
2. 玩家拖动超出范围时不计入成功；
3. 玩家可重新开始当前小关卡；
4. 失败不扣除关键道具。

### 17.6 验收标准

纹样对齐验收：

1. 底层纹样与拓印纸能显示；
2. 玩家能调整拓印纸；
3. 达到阈值后成功；
4. 成功后获得拓片。

关键点描摹验收：

1. 关键点显示清楚；
2. 玩家能按顺序完成；
3. 错误顺序有提示；
4. 完成后获得拓片。

1. 可修复银饰部件列表；
2. 当前部件图片或占位图；

## F-16 敲银时机小游戏

### 19.1 功能目标

敲银时机小游戏用于表现“手工修复”的操作感。拓印负责获得纹样材料，敲银负责使用纹样材料完成银饰加工，两者在功能和操作上保持区分。

### 19.2 输入

1. 空格：敲银确认；
2. 鼠标点击：敲银确认，可选；

1. 敲银进度条；
2. 移动指针；
3. 银色判定区；
4. 命中反馈；
5. 未命中反馈；
6. 银饰亮度变化；
7. 修复完成提示。

### 19.4 小游戏参数

| 参数 | 说明 |
|---|---|
| required_hits | 需要命中次数。 |
| pointer_speed | 指针移动速度。 |
| target_zone_width | 判定区宽度。 |
| misses | 未命中次数。 |
| max_attempts | 最大尝试次数，可选。 |

### 19.5 难度建议

| 部件 | required_hits | 判定区 | 指针速度 |
|---|---:|---|---|
| 银耳环 | 1 | 宽 | 慢 |
| 银手镯 | 2 | 中 | 中 |

### 19.6 主流程

6. 命中则 hits + 1，银饰亮度增加；
7. 未命中则 misses + 1，显示轻微失败反馈；
8. hits 达到 required_hits 后修复成功；
9. 系统更新银饰部件状态和银光值；
10. 返回工坊界面或 PLAYING。

2. 未命中可轻微降低 repair_score 或 silver_score；
3. 连续失败后银匠师傅给出提示；
4. 关键材料不因失败被扣除；
5. 玩家可以重新尝试。

### 19.8 验收标准

1. 敲银界面能显示；
2. 指针能移动；
3. 判定区能显示；
4. 玩家输入能被正确识别；
5. 命中和未命中有不同反馈；
6. 成功后银饰部件点亮；
7. 失败不导致任务卡死；
8. 成功后套装进度、图鉴和任务能更新。

4. 图鉴解锁和任务、物品、纹样寻源、银饰修复相关；

6. 纹样图鉴和银饰图鉴能区分显示。

## F-18 银光值与结局评价系统

### 21.1 功能目标

通过银光值、银饰套装进度、图鉴进度、隐藏收集和支线完成情况评价玩家表现，解决“完成后就没有追求”的问题。

### 21.2 核心数据

| 数据 | 说明 |
|---|---|
| silver_score | 银光值。 |
| repaired_parts | 已修复银饰部件。 |
| repaired_count | 修复数量。 |
| codex_count | 图鉴解锁数量。 |
| hidden_count | 隐藏银纹数量。 |
| side_task_count | 支线完成数量。 |
| repair_misses | 敲银失败次数。 |
| pattern_quality | 拓片质量，可选。 |

### 21.3 银光值变化规则

建议规则：

| 行为 | 银光值变化 |
|---|---:|
| 修复一个银饰部件 | +10 |
| 获得清晰拓片 | +3 |
| 解锁一个图鉴 | +2 |
| 完成支线任务 | +5 |
| 找到隐藏银纹 | +5 |
| 敲银失败一次 | -1 |
| 纹样路径多次走错 | -1，可选 |

银光值变化应温和，不应导致玩家无法通关。

### 21.4 套装进度规则

1. 每修复一个新银饰部件，repaired_count + 1；
2. 已修复部件不能重复计数；
3. HUD 显示“节庆银饰套装：x / 4”；
4. 展示版本可要求 x ≥ 1 或 x ≥ 2 进入结局；
5. 完整版本可要求 x ≥ 4 达成完整守护者结局。

### 21.5 结局类型

| 结局 | 条件建议 | 表现 |
|---|---|---|
| 普通结局 | 完成最低修复条件，银光值较低或图鉴较少 | 银饰发亮，但展示较简单。 |
| 良好结局 | 修复 2 到 3 个部件，图鉴较完整 | 节庆广场灯光亮起，NPC 表示认可。 |
| 完整守护者结局 | 修复完整套装，银光值高，图鉴和支线较完整 | 银饰套装闪耀，获得“银纹守护者”称号。 |

### 21.6 节庆展示流程

1. 玩家进入 festival 场景；
2. 玩家与节庆主持人对话；
3. 系统检查 repaired_count 是否达到最低要求；
4. 若未达到要求，提示继续修复；
5. 若达到要求，进入 FESTIVAL_SHOW；
6. 系统播放银饰展示反馈；
7. 系统计算结局评价；
8. 系统进入 ENDING；
9. 结局界面显示称号、套装进度、银光值、图鉴进度。

### 21.7 验收标准

1. 银光值能初始化；
2. 修复、图鉴、支线、隐藏收集能影响银光值；
3. HUD 或结局界面能显示银光值；
4. 套装进度能正确更新；
5. 结局评价与玩家行为存在对应关系；
6. 结局界面可重新开始或退出。

## F-19 UI/HUD 系统

### 22.1 功能目标

提供游戏中必要的信息提示，帮助玩家理解当前目标、纹样寻源进度、银饰套装进度和操作方式。

### 22.2 HUD 内容

主游戏界面应显示：

1. 当前任务；
2. 节庆银饰套装进度；
3. 银光值；
4. 图鉴进度；
5. 当前交互提示；
6. 简短反馈信息。

### 22.3 HUD 示例

```text
当前任务：寻找山纹来源
节庆银饰套装：2 / 4
银光值：75
图鉴进度：4 / 8
```

### 22.4 纹样寻源 HUD

source_hunt：

```text
水波纹线索：2 / 3
继续寻找水边的纹样线索
```

path_maze：

```text
山纹路径：3 / 5
请沿着银色线条继续前进
```

align_rubbing：

```text
拓印重合度：86%
继续微调拓印纸位置
```

point_trace：

```text
关键点：2 / 5
请连接下一个发光点
```

### 22.5 交互提示规则

1. 靠近 NPC：按 E 对话；
2. 靠近物品：按 E 拾取；
3. 靠近纹样寻源点：按 E 进入纹样寻源；
4. 靠近线索点：按 E 记录线索；
5. 靠近修复台：按 E 修复银饰；
6. 靠近节庆展示台：按 E 展示银饰。

### 22.6 UI 显示原则

1. UI 不遮挡玩家和关键对象；
2. 任务提示文字简短；
3. 交互提示只在必要时出现；
4. 图鉴和背包打开时暂停玩家移动；
5. 失败提示应温和，避免打击玩家；
6. 纹样小关卡 UI 应突出当前目标和完成进度。

### 22.7 验收标准

1. HUD 能显示当前任务；
2. HUD 能显示银饰套装进度；
3. HUD 能显示银光值或结局相关完成度；
4. 交互提示能根据对象变化；
5. 背包和图鉴界面能打开关闭；
6. UI 不影响玩家基本操作；
7. 任务完成后 HUD 能更新。

## F-20 音效与视觉反馈系统

### 23.1 功能目标

通过音效和视觉反馈强化玩家操作感、纹样获得感、敲银修复感和结局完成感。

### 23.2 音效列表

| 音效 | 触发时机 |
|---|---|
| button_click | 点击菜单按钮。 |
| dialogue_pop | 打开对话。 |
| item_pickup | 拾取物品。 |
| clue_record | 记录自然线索。 |
| checkpoint_light | 点亮纹样检查点。 |
| pattern_complete | 完成纹样寻源。 |
| hammer_hit | 敲银命中。 |
| hammer_miss | 敲银未命中。 |
| repair_success | 银饰修复成功。 |
| codex_unlock | 图鉴解锁。 |
| ending_show | 节庆展示。 |

### 23.3 视觉反馈列表

1. 可拾取物品带轻微闪光；
2. 纹样寻源点发光；
3. 自然线索点记录后变暗或打勾；
4. 纹样检查点点亮；
5. 纹样路径逐段发光；
6. 完整纹样合成；
7. 获得拓片弹窗；
8. 敲银命中时银光闪烁；
9. 修复成功时银饰从暗淡变为发亮；
10. 图鉴解锁弹窗；
11. 节庆广场灯光亮起。

### 23.4 异常处理

1. 音效文件缺失时跳过播放，不崩溃；
2. 图片缺失时使用占位图形；
3. 动画资源缺失时使用静态图；
4. 反馈对象不存在时只显示文本提示。

### 23.5 验收标准

1. 至少实现拾取、纹样完成、修复成功三类反馈；
2. 反馈不影响游戏流程；
3. 玩家能明确知道操作是否成功；
4. 缺少音效素材时游戏不崩溃；
5. 敲银命中和未命中反馈可区分。

## F-21 本地存档系统（可选）

### 24.1 功能目标

记录玩家当前进度，方便继续游戏。该功能为优秀扩展，不是最低必需功能。

### 24.2 存档内容

存档文件建议保存为 JSON，字段包括：

1. current_scene；
2. player_position；
3. completed_tasks；
4. active_task；
5. inventory；
6. unlocked_codex；
7. completed_patterns；
8. collected_clues；
9. repaired_parts；
10. silver_score；
11. completed_side_tasks。

### 24.3 保存触发

可选保存方式：

1. 切换场景时自动保存；
2. 完成任务时自动保存；
3. 完成纹样寻源时自动保存；
4. 修复银饰部件时自动保存；
5. 玩家在菜单中点击保存；
6. 进入结局时保存完成状态。

### 24.4 读取流程

1. 玩家点击继续游戏；
2. 系统读取 save.json；
3. 系统恢复场景、坐标、任务、背包、图鉴、纹样、银饰部件和银光值；
4. 如果存档不存在，则提示开始新游戏。

### 24.5 异常处理

1. 存档文件不存在时开始新游戏；
2. 存档字段缺失时使用默认值；
3. 存档损坏时提示重新开始；
4. 存档版本不兼容时提供重置选项。

### 24.6 验收标准

1. 优秀版本可以保存进度；
2. 读取后任务和背包不丢失；
3. 已完成纹样和已修复部件不丢失；
4. 银光值和图鉴状态可恢复；
5. 存档异常不会导致游戏崩溃。

## 26. 功能之间的数据流

### 26.1 主线数据流

```text
NPC 对话
  ↓
任务状态更新
  ↓
地图目标开放
  ↓
自然线索 / 纹样寻源点开放
  ↓
完成 source_hunt 或 path_maze
  ↓
获得纹样拓片
  ↓
背包数据更新
  ↓
图鉴解锁
  ↓
收集银饰材料
  ↓
工坊修复条件满足
  ↓
敲银小游戏
  ↓
银饰部件状态变化
  ↓
套装进度与银光值更新
  ↓
节庆展示结局评价
```

### 26.2 纹样寻源数据流

```text
PatternSourcePoint
  ↓
读取 pattern_id / challenge_type / reward_item
  ↓
进入对应挑战
  ↓
完成线索收集或检查点路径
  ↓
Inventory.add_item(reward_item)
  ↓
Codex.unlock(pattern_codex_id)
  ↓
TaskManager.on_pattern_completed(pattern_id)
  ↓
ScoreManager.add_pattern_score(pattern_quality)
```

### 26.3 工坊修复数据流

```text
RepairTable
  ↓
选择 silver_part
  ↓
读取 required_items / required_patterns
  ↓
Inventory.has_item(...)
  ↓
进入 Hammer MiniGame
  ↓
命中 required_hits
  ↓
SilverPart.state = bright
  ↓
ScoreManager.add_repair_score(...)
  ↓
Codex.unlock(silver_part_codex)
  ↓
TaskManager.on_part_repaired(part_id)
```

### 26.4 结局评价数据流

```text
repaired_parts
+ silver_score
+ unlocked_codex
+ hidden_collectibles
+ completed_side_tasks
+ repair_performance
  ↓
ScoreManager.calculate_ending()
  ↓
EndingScreen 显示称号、完成度和评价
```

## 29. 给 AI/Codex 的功能实现顺序

### 阶段一：基础工程

请实现项目基础结构，包括 main.py、settings.py、Game 类、主循环、窗口创建和主菜单。不要实现其他功能。

### 阶段二：地图与摄像机

请实现 Tiled 地图加载、视窗 800×600、瓦片 32×32、摄像机跟随和摄像机越界限制。先使用一张测试地图。

### 阶段三：玩家与碰撞

请实现 Player 类、上下左右移动、地图边界限制和 collision 层碰撞检测。

### 阶段四：对象层读取

请读取 Tiled 对象层，生成 NPC、Item、PatternSourcePoint、CluePoint、Checkpoint、Trigger 等对象。不要硬编码对象坐标。

### 阶段五：NPC 与对话

请实现 NPC 类、交互提示、按 E 对话、对话框和根据任务状态变化的对话文本。对话需要能提供纹样来源线索。

### 阶段六：物品与背包

请实现物品拾取、背包数据记录、按 I 打开背包界面。背包需要能记录银饰碎片、银丝材料和纹样拓片。

### 阶段七：任务系统

请实现主线任务 main_01 到 main_08，支持任务状态更新、HUD 当前任务显示和银饰套装进度记录。

### 阶段八：纹样寻源系统一：source_hunt

请实现 source_hunt 自然场景寻源：读取 clue_points，支持线索记录、线索数量统计、完成后获得纹样拓片、更新背包/任务/图鉴。

### 阶段九：纹样寻源系统二：path_maze

请实现 path_maze 走纹样迷宫：加载纹样小地图，读取 checkpoint_points，按顺序点亮检查点，完成后获得拓片并返回原场景。

### 阶段十：工坊修复与敲银小游戏

请实现 RepairMiniGame：选择银饰部件、检查材料与拓片、进入敲银时机判定、命中后点亮银饰部件、更新任务、图鉴和银光值。

### 阶段十一：图鉴、银光值与结局

请实现银饰图鉴、纹样图鉴、银光值计算、节庆广场展示和结局评价界面。

### 阶段十二：扩展玩法与展示优化

请补充 align_rubbing 纹样对齐或 point_trace 关键点描摹，增加音效、视觉反馈、错误处理、调试信息和完整流程测试。

## 30. 最终说明

本功能规格说明书将《银纹秘境：彝饰守护者》的新版游戏策划与软件需求进一步细化为可开发、可测试、可验收的功能模块。

本项目最终应形成一个完整的 2D 非战斗型文化修复探索游戏 Demo。玩家可以在大于视窗的地图中移动，和 NPC 对话，收集银饰碎片，理解纹样来源，进入纹样空间走出纹样路径，获得纹样拓片，返回工坊完成敲银修复，点亮节庆银饰套装，解锁图鉴，并在节庆广场根据完成度获得结局评价。

从功能规格角度看，本项目的核心不是功能数量越多越好，而是要做到：

1. 主线流程完整；
2. 地图与视窗规范；
3. 对象层读取清晰；
4. 面向对象结构合理；
5. 任务、背包、纹样寻源、修复、图鉴、结局形成闭环；
6. 文化主题与玩法强绑定；
7. 拓印和敲银功能区分明确；
8. 每个功能都有明确输入、输出、状态变化和验收标准。

新版 FSD 的核心变化是：

1. 将 F-11 从旧版“纹样拓印系统”升级为“纹样寻源系统”；
2. 新增 F-12 走纹样迷宫系统；
3. 新增 F-13 自然场景寻源系统；
4. 新增 F-14 纹样对齐与关键点描摹扩展；
5. 将 F-12 旧版修复小游戏升级为“银饰工坊修复 + 敲银时机小游戏”；
6. 新增银饰套装进度、银光值和结局评价功能；
7. 将对象层读取扩展到纹样寻源点、线索点和检查点；
8. 将 AI/Codex 实现顺序调整为更适合分阶段开发的版本。
